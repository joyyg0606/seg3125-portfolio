About Me

Learning UI design by building each assignment

Hi, my name is Joy Yeung. I am a Software Engineering student at the University of Ottawa. Before SEG3125, I usually thought more about whether my code worked than how the interface felt to use.

During the course, I focused on readable layouts, clear feedback, simple navigation, and matching each page to a real user goal. The projects below let me practise a different type of interaction each time.

How I Work

The process I followed for the course

01

Write the User Goal

I first write down what the user should be able to finish and what information they need.

02

Plan the Main Flow

I split the task into smaller steps and decide which controls and feedback should be visible.

03

Build a Working Version

I create the prototype with React, Bootstrap, CSS, and straightforward JavaScript.

04

Test What Feels Unclear

I check labels, feedback, spacing, and the places where a user might hesitate or make a mistake.

Final Reflection

What changed from the first assignment

My first portfolio focused mostly on basic layout and appearance. Later assignments made me think more about user goals, system feedback, cognition, verbal communication, usability, and localization.

The last dashboard brought several ideas together. It needed clear charts, useful controls, bilingual wording, localized numbers, and enough explanation for the user to understand the data.

Final portfolio includes

  • Four working interactive projects
  • Project previews that match each site
  • Responsive layouts for smaller screens
  • Clear navigation back to the portfolio
  • An English and French dashboard
